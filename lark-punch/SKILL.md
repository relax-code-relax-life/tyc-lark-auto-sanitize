# 飞书打卡（Lark Auto Punch）

## 触发场景
- 用户说「飞书打卡」、「帮我打卡」、「lark 打卡」、「clock in」
- 用户要求执行上班或下班打卡

## 执行步骤

### 1. 在本机执行打卡脚本（不要用 nodes.run）

**必须使用本机 exec** 执行脚本（此 macOS 节点不支持 system.run.prepare，nodes.run 会被拒绝）：

```bash
/Users/wangwl.net/tyc/lark_auto/exec.sh 2>&1
```

> `exec.sh` 会自动加载环境变量、进入项目目录并调用 `main.py`，根据当前时间自动判断是上班还是下班打卡。

**建议超时**：120s。

### 2. 返回日志（原样）

将上一步命令的**完整输出**（stdout + stderr）原文发送给用户，不做任何截断或摘要。

## 注意事项

- 脚本内置时间判断逻辑：< 14:00 执行上班打卡，>= 14:00 执行下班打卡
- 若今日在跳过日期列表中，脚本会提前退出并在日志中说明
- 若脚本执行失败（非零退出码），同样将完整错误日志发送给用户并提示检查设备连接/ADB/Appium 状态
